# Roadmap: مدير ملفات المحاكم (Mahkama Dossier Manager)

## Overview

تطوير تطبيق محلي متكامل لإدارة وتتبع ملفات المحاكم بالاعتماد على محرك معالجة ملفات Excel بلغة Python، وواجهة ويب تفاعلية RTL باللغة العربية متطابقة مع الهوية البصرية لبوابة محاكم الرسمية.

## Phases

- [x] **Phase 1: Core Parsing & Logic Engine** - تطوير محرك القراءة ومعالجة البيانات وإعادة التسمية بالباك اند.
- [x] **Phase 2: RTL Web Interface & Settings Panel** - تطوير واجهة الويب العربية ولوحة التحكم وتعديل الإعدادات.
- [x] **Phase 3: Internal Portal Automation** - بناء سكريبت لجلب ملفات السجل العام من بوابة محاكم الداخلية.
- [x] **Phase 4: Web UI Automation Integration** - دمج سكريبت الأتمتة في واجهة الويب بخياري (تلقائي / يدوي).

## Phase Details

### Phase 4: Web UI Automation Integration
**Goal**: تطوير شاشة بداية (Landing Page) تخير المستخدم بين رفع الملفات يدوياً أو جلبها تلقائياً للسنة المطلوبة، مع عرض نفس جدول البيانات النهائي.
**Depends on**: Phase 2, Phase 3
**Requirements**: 
- صفحة بداية بخيارين (يدوي / تلقائي).
- مسار (Route) في Flask لتشغيل سكريبت `sync_dossiers.py` في الخلفية.
- عرض شريط تحميل (Progress) أو شاشة انتظار للمستخدم.
- تحويل المستخدم لجدول البيانات وعرض الملفات التي تم تنزيلها.

### Phase 3: Internal Portal Automation
**Goal**: أتمتة الدخول لموقع المحكمة الداخلي (10.250.1.26) وتحميل ملفات السجل العام لجميع الفئات وتخزينها تلقائياً.
**Depends on**: Phase 1
**Requirements**: 
- تسجيل الدخول تلقائياً (ndmini/password).
- التوجه لصفحة السجلات واختيار السنة.
- استخراج وتحميل جميع الملفات التي تحمل رابط "تحميل السجل العام".
**Success Criteria** (what must be TRUE):
  1. يمكن للبرنامج تسجيل الدخول للبوابة وتجاوز صفحة البداية.
  2. يستطيع البرنامج تحديد جميع روابط التحميل للسنة المختارة وحفظها بصيغة Excel داخل مجلد التحميلات.

### Phase 1: Core Parsing & Logic Engine
**Goal**: تطوير محرك الباك اند لقراءة ملفات Excel وحساب الآجال وإعادة تسمية الملفات.
**Mode**: mvp
**Depends on**: Nothing (first phase)
**Requirements**: FILE-01, FILE-02, FILE-03, RENAME-01, RENAME-02, CALC-01, CALC-02
**Success Criteria** (what must be TRUE):
  1. يمكن للنظام قراءة ملفات Excel واستخراج رمز الفئة وتاريخ التسجيل بنجاح.
  2. يقوم المحرك بإعادة تسمية الملفات تلقائياً وحمايتها من أخطاء الإغلاق/القفل.
  3. يتم حساب الأيام المتبقية بشكل دقيق لكل فئة بناءً على قواعد التواريخ.
**Plans**: 2 plans

Plans:
- [x] 01-01: تطوير وحدة قراءة وتحليل ملفات Excel وحساب الآجال وقراءة الإعدادات.
- [x] 01-02: تطوير نظام مسح المجلدات وإعادة التسمية التلقائية وبناء خادم Flask.

### Phase 2: RTL Web Interface & Settings Panel
**Goal**: بناء واجهة ويب عربية متكاملة ولوحة الإعدادات والربط النهائي.
**Mode**: mvp
**Depends on**: Phase 1
**Requirements**: DASH-01, DASH-02, SETT-01, SETT-02
**Success Criteria** (what must be TRUE):
  1. تظهر واجهة الويب باللغة العربية RTL وتصميم يماثل بوابة محاكم.
  2. تعرض الواجهة جدول الملفات مرتباً حسب درجة الاستعجال مع مؤشرات الألوان (أحمر، برتقالي، أخضر).
  3. تتيح لوحة الإعدادات تعديل الآجال وحفظها في settings.json وتحديث الحسابات فوراً.
**Plans**: 2 plans

Plans:
- [x] 02-01: تصميم صفحة الجدول الرئيسي (Dashboard) باللغة العربية ونمط بوابة محاكم.
- [x] 02-02: بناء صفحة الإعدادات وربط حفظ البيانات (settings.json) مع الباك اند وإتمام التجربة الكاملة.
